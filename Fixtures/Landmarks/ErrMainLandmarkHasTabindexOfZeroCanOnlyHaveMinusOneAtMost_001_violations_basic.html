Main Landmark Inappropriate Tabindex - Violations

Article Content

This main landmark has tabindex="0" which inappropriately adds it to the tab order.

Keyboard users will tab to the landmark itself, which is confusing and redundant since they can already tab to interactive elements within it.

Dashboard Content

This main landmark has a positive tabindex, which is even worse than tabindex="0".

Positive tabindex values disrupt the natural reading order and create unpredictable navigation.

Settings

Product Catalog

High positive tabindex values create an unpredictable and confusing tab order.

Landmarks should not be in the tab order. Use tabindex="-1" only if programmatic focus is needed (e.g., skip links), never tabindex="0" or positive values.
