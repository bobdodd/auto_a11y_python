ErrTabindexColorChangeOnly - Violation Examples

This page contains elements with tabindex that use ONLY color changes for focus indicators. All should generate errors per WCAG 1.4.1 Use of Color.

Color-Only Focus Indicators

Violation 1: Background Color Change Only

Focus changes background from white to light blue - no outline/border/shadow

Violation 2: Text Color Change Only

Focus changes text from black to blue - violates Use of Color

Violation 3: Border Color Change Only

Border color changes but width stays same - no structural change

Violation 4: Multiple Color Changes

Both background and text color change - still color-only indicator

Violation 5: Border + Background Color

Border and background colors change - no outline/box-shadow added

Expected Test Results:

  • Error Code: ErrTabindexColorChangeOnly
  • Count: 5 violations detected
  • WCAG: 1.4.1 Use of Color (Level A)
  • Why: Focus indicated by color alone, without structural changes
  • Details:
    • Violation 1: Background color change only
    • Violation 2: Text color change only
    • Violation 3: Border color change (same width)
    • Violation 4: Multiple colors change (still color-only)
    • Violation 5: Border + background color change
  • Impact: Users with color blindness or monochrome displays cannot perceive focus
  • Fix: MUST add structural indicator: outline, box-shadow, or border width change
