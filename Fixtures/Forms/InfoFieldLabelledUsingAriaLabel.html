Test Coverage Summary

This fixture includes the following test scenarios for InfoFieldLabelledUsingAriaLabel:

  • Basic aria-label usage: Text inputs, email inputs, and textareas with only aria-label
  • Search and filter fields: Common UI patterns that often use aria-label
  • Inline form fields: Compact layouts where visible labels might be omitted
  • Dynamic fields: Context-dependent fields using aria-label
  • Mixed labeling: Fields with aria-label combined with other labeling methods
  • Complex controls: Multi-select, range, and color inputs with aria-label
  • Proper labeling: Counter-examples that should NOT trigger the info message

Expected Results: The accessibility checker should identify all fields in sections 1-6 that use aria-label and report them with the InfoFieldLabelledUsingAriaLabel message, while properly labeled fields in section 7 should not trigger this info message.
