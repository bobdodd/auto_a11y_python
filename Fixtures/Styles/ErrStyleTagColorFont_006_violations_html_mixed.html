Mixed Properties in Style Tag

⚠️ This is a Violation (Mixed Case)

The style tag mixes CSS variables with hard-coded colors on html.

What's in the Style Tag?

<style>
    html {
        --primary-color: #007bff;  ✅ CSS variable (allowed)
        background-color: #ffffff;  ❌ Hard-coded color (not allowed)
        --spacing: 8px;             ✅ CSS variable (allowed)
    }
</style>

Why This Should Be Flagged

  • Contains a hard-coded background-color property
  • The CSS variables alone would be fine, but the mix is problematic
  • ALL properties must be allowed for the style tag to pass
  • The background-color prevents user customization

Test Logic

  1. Check if style tag contains a rule targeting html
  2. Extract all CSS declarations from the html rule
  3. Check if ALL declarations are allowed (CSS variables or color-scheme)
  4. If ANY declaration is NOT allowed → Flag as violation
  5. If ALL declarations are allowed → Skip (no violation)

Test Expectation

Expected Result: The first style tag SHOULD be flagged as ErrStyleTagColorFont because it contains a hard-coded color property mixed with CSS variables.

How to Fix

Convert the hard-coded color to a CSS variable:

<style>
    html {
        --primary-color: #007bff;
        --bg-color: #ffffff;
        --spacing: 8px;
    }
    body {
        background-color: var(--bg-color);
    }
</style>
