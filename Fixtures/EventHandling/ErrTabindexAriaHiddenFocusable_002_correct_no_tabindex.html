Why This Is Correct

Decorative SVG icons can and should use aria-hidden="true" to hide them from screen readers when the button text provides the accessible name.

However, these decorative elements should never have tabindex. The button itself is already focusable and clickable - child elements don't need their own keyboard focus.

With this pattern:

  • Keyboard users tab to the button (not the SVG)
  • Screen reader users hear "Search, button" (not the SVG details)
  • The button's focus indicator is visible to all users
  • No conflicting ARIA semantics
