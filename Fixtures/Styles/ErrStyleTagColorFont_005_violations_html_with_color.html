Hard-Coded HTML Colors in Style Tag

⚠️ This is a Violation

The <style> tag sets hard-coded colors on the html element.

What's Wrong?

The first style tag contains:

<style>
    html {
        background-color: #f5f5f5;
        color: #333333;
    }
</style>

Why This Should Be Flagged

  • Sets actual color properties (not CSS variables)
  • Prevents users with accessibility needs from customizing colors
  • Breaks high contrast modes
  • Ignores user preferences
  • Makes it difficult for users to override with their own stylesheets

Test Expectation

Expected Result: The first style tag SHOULD be flagged as ErrStyleTagColorFont because it contains hard-coded color properties on the html element.

How to Fix

Use CSS variables instead:

<style>
    html {
        --bg-color: #f5f5f5;
        --text-color: #333333;
    }
    body {
        background-color: var(--bg-color);
        color: var(--text-color);
    }
</style>
