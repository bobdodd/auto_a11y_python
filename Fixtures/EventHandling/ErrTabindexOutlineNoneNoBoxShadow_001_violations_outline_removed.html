ErrTabindexOutlineNoneNoBoxShadow - Violation Examples

This page contains elements with tabindex that have outline:none (or equivalent) without a box-shadow alternative. All should generate errors.

Outline Removed Without Alternative

Violation 1: outline: none

Explicitly removes outline with no alternative focus indicator

Violation 2: outline: 0

Sets outline to 0, which is equivalent to 'none'

Violation 3: outline-style: none

Removes outline by setting style to none

Violation 4: outline-width: 0

Makes outline invisible by setting width to 0

Violation 5: outline: none with border-only change

Border changes color but no box-shadow alternative (border alone isn't sufficient)

Expected Test Results:

  • Error Code: ErrTabindexOutlineNoneNoBoxShadow
  • Count: 5 violations detected
  • WCAG: 2.4.7 Focus Visible (Level AA)
  • Why: Outline removed without providing box-shadow alternative
  • Details:
    • Violation 1: outline: none - no alternative
    • Violation 2: outline: 0 - equivalent to none
    • Violation 3: outline-style: none - outline suppressed
    • Violation 4: outline-width: 0 - invisible outline
    • Violation 5: outline: none with only border change (insufficient)
  • Impact: Keyboard users cannot see focus location - critical accessibility failure
  • Fix: If removing outline, MUST provide box-shadow as alternative indicator
