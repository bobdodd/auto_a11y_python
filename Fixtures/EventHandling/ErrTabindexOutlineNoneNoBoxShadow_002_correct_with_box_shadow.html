ErrTabindexOutlineNoneNoBoxShadow - Correct Examples

This page contains elements with tabindex that either have box-shadow alternatives when outline is removed, or keep the default outline. None should generate errors.

Proper Focus Indicators

Correct 1: Box-Shadow Alternative

outline: none BUT provides box-shadow: 0 0 0 3px #0066cc

Correct 2: Multi-Layer Box-Shadow

outline: 0 BUT provides multiple box-shadow layers for depth

Correct 3: Inset Box-Shadow

outline-style: none BUT provides inset box-shadow alternative

Correct 4: Spread Box-Shadow

outline-width: 0 BUT provides glowing box-shadow with spread

Correct 5: Browser Default Outline

No explicit outline removal - uses browser's default focus indicator

Expected Test Results:

  • Error Code: None (negative test)
  • Count: 0 errors
  • WCAG: 2.4.7 Focus Visible (Level AA)
  • Why: Either box-shadow alternative provided when outline removed, or default outline kept
  • Details:
    • Correct 1-4: outline removed BUT box-shadow alternative provided
    • Correct 5: No outline removal - browser default preserved
  • Best Practice: When removing outline for aesthetic reasons, ALWAYS provide box-shadow as alternative
