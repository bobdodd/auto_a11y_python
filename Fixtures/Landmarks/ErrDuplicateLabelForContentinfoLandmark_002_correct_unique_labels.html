Recommended Pattern: Single ContentInfo

Most pages should have only one footer/contentinfo landmark, which doesn't require a label. This is demonstrated by the single footer element pattern shown above (before the additional labeled contentinfo examples).

Best Practice: Use a single <footer> element per page. If you need multiple footer-like sections, consider using complementary landmarks (<aside>) for supplementary sections and reserve contentinfo for the main page footer.
