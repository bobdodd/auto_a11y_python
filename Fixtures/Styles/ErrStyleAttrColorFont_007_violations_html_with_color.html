Hard-Coded Colors on HTML Element

⚠️ This is a Violation

The <html> element has inline color styles that should be flagged.

What's Wrong With This Page?

The HTML element has this inline style:

<html style="background-color: #f0f0f0; color: #333;">

This is problematic because:

  • Prevents user customization - Users with accessibility needs cannot override these colors
  • Breaks high contrast modes - Inline styles have high specificity and override user stylesheets
  • Ignores user preferences - Users who configured custom colors in their browser are forced to use these
  • Not responsive to system settings - Doesn't respect dark mode or other OS-level accessibility settings

Difference from CSS Variables

This is different from CSS variables or color-scheme:

Type Example Should Flag? Reason
CSS Variables --primary: #007bff; ❌ No Defines reusable values, easily overridden
Color Scheme color-scheme: dark light; ❌ No Accessibility feature for dark mode support
Hard-Coded Colors background-color: #f0f0f0; ✅ Yes Prevents user customization

Who This Affects

Hard-coded colors on the HTML element impact:

  • Users with low vision who need high contrast colors
  • Users with photophobia who need dark mode
  • Users with color blindness who configured custom color schemes
  • Users with dyslexia who benefit from specific background colors
  • Older users with age-related vision changes who need custom colors

How to Fix

Instead of inline styles on <html>, use:

  1. External CSS in a stylesheet (easier for users to override with !important)
  2. CSS variables if you need to set design tokens
  3. color-scheme property for dark mode support
  4. Media queries to respect user preferences like prefers-color-scheme
