ErrTabindexSingleSideBoxShadow - Violation Examples

This page contains elements with tabindex that use single-sided box-shadow for focus indicators. All should generate errors per CR 5.2.4 requirements.

Single-Sided Box-Shadow Focus Indicators

Violation 1: Left Side Only

box-shadow: -3px 0 0 0 #0066cc (negative X offset only)

Violation 2: Right Side Only

box-shadow: 3px 0 0 0 #cc0000 (positive X offset only)

Violation 3: Top Side Only

box-shadow: 0 -3px 0 0 #006600 (negative Y offset only)

Violation 4: Bottom Side Only

box-shadow: 0 3px 0 0 #cc6600 (positive Y offset only)

Violation 5: Left Side with Blur

box-shadow: -4px 0 8px 0 rgba(...) - blur doesn't fix single-sided issue

Expected Test Results:

  • Error Code: ErrTabindexSingleSideBoxShadow
  • Count: 5 violations detected
  • WCAG: CR 5.2.4 (Common Requirement for focus indicators)
  • Why: Single-sided focus indicators can be cropped by zoom/magnification
  • Details:
    • Violation 1: Left side only (-3px X offset)
    • Violation 2: Right side only (+3px X offset)
    • Violation 3: Top side only (-3px Y offset)
    • Violation 4: Bottom side only (+3px Y offset)
    • Violation 5: Left side with blur (still single-sided)
  • Impact: Screen magnifier users may not see indicator if it's on cropped side
  • Fix: Use box-shadow: 0 0 0 Npx (uniform on all sides) or outline instead
