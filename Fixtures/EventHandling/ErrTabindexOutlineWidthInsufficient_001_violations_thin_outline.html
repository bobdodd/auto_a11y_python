ErrTabindexOutlineWidthInsufficient - Violation Examples

This page contains elements with tabindex that have outline widths less than 2px. All should generate errors per WCAG 2.4.11 (AAA).

Insufficient Outline Width

Violation 1: 1px Outline

outline: 1px solid #0066cc - below 2px AAA requirement

Violation 2: 0.5px Outline

outline: 0.5px solid #cc0000 - extremely thin, hard to perceive

Violation 3: 1.5px Outline

outline: 1.5px solid #006600 - still below 2px threshold

Violation 4: "thin" Keyword

outline: thin solid #cc6600 - typically renders as 1px

Violation 5: 1px with Offset

outline: 1px with outline-offset: 3px - offset doesn't fix width issue

Expected Test Results:

  • Error Code: ErrTabindexOutlineWidthInsufficient
  • Count: 5 violations detected
  • WCAG: 2.4.11 Focus Appearance (Level AAA)
  • Why: Outline width < 2px fails AAA focus appearance requirements
  • Details:
    • Violation 1: 1px outline
    • Violation 2: 0.5px outline (extremely thin)
    • Violation 3: 1.5px outline
    • Violation 4: "thin" keyword (~1px)
    • Violation 5: 1px with offset (offset doesn't help width)
  • Impact: Users with low vision may struggle to perceive thin focus indicators
  • Fix: Use outline-width ≥ 2px for AAA conformance (or ≥ 3px for better visibility)
