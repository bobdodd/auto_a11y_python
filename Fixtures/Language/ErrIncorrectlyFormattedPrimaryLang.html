Test Fixture: Incorrectly Formatted Primary Language Code

Current Document Issues

❌ This document has lang="INVALID_LANG_FORMAT" on the html element

This should trigger the ErrIncorrectlyFormattedPrimaryLang error because:

  • The language code format is completely invalid
  • It doesn't follow BCP 47 standards
  • Screen readers won't recognize this format

Common Invalid Formats That Should Trigger Errors

  • lang="ENGLISH" - Full language name instead of code
  • lang="en_US" - Using underscore instead of hyphen
  • lang="EN-us" - Incorrect case formatting
  • lang="english" - Lowercase full name
  • lang="123" - Numeric values
  • lang="en-" - Incomplete subtag
  • lang="en-USA" - Three-letter country code
  • lang="" - Empty string
  • lang=" en " - Extra whitespace
  • lang="en,fr" - Multiple languages (comma-separated)

Valid Formats (For Reference)

✅ These would be correct formats:

  • lang="en" - Simple language code
  • lang="en-US" - Language with region
  • lang="fr-CA" - French (Canada)
  • lang="zh-Hans" - Chinese (Simplified script)
  • lang="de-DE" - German (Germany)
  • lang="es" - Spanish

Impact on Users

When the primary language is incorrectly formatted:

  • Screen readers may not pronounce content correctly
  • Text-to-speech engines won't use proper pronunciation rules
  • Browser translation features may not work
  • Voice navigation software may malfunction
  • Users who rely on assistive technology experience degraded content
